1. INTRODUCTION

Javascript (à ne pas confondre avec java) est un langage de programmation dynamique et non typé (c'est à dire qu'une variable peut contenir n'importe quel type de valeur) utilisé le plus souvent pour le coté client mais peut etre utilisé aussi coté server (nodeJS). Javascript est communément appelé JS, elle est sensible à la casse, cela signifie que le langage considère les majuscules différents des miniscules. Les mots clés en JS sont tous en miniscules.
Javascript a été créé en 1995 par Brendan Eich et intégré au navigateur Netscape navigator.
Les fichiers JS seuls ne peuvent pas etre exécutés par le navigateur, il faut donc les integrer à une page html.

Un commentaire est donc une indication qui ne sera pas lu par les navigateurs, donc invisible pour les utilisateurs.

3. Les variables

3.1 Définition

Comme vu précédemment en algorithmie, une variable est un conteneur servant à stocker des informations de manière temporaire. le propre d'une variable comme sont nom l'indique est donc de varier, c'est à dire de pouvoir contenir plusieurs valeurs au fil du temps. C'est bien cet aspect qui les rendent utiles cependant il faut bien differencier la variable qu'on appelle par son nom de la valeur qu'elle contienne. Il n'y a pas d'égalité entre la variable et sa valeur meme si la syntaxe porte à confusion.

3.2 Déclaration

Pour déclarer une variable en JS, on utilise les mots clés let ou var suivi du nom de la variable.
le nom d'une variable doit obligatoirement commencer par une lettre ou un underscore (_) et ne doit jamais commencer par un chiffre.
Le nom d'une variable ne doit pas contenir d'espace, mais peut contenir des lettres, des chiffres, des tirets mais pas de caractères spéciaux.
En JS, il recommandé d'utiliser le camelCase pour nommer les variables.
exemple: let puitDuFou= "parc d'attraction";

3.3 Les types de variables

En javascript, on distingue 3 types de variables.
Numbers: ce sont les variables de type numérique
Strings: pour les chaines de caractères
Boolean : pour le true or false

3.4 les types de valeurs

En JS, contrairement à la plus part des langages de programmation, nous n'avous pas besoin de préciser le type de variable à la déclaration. Le JS étant un langage non typé c'est la valeur qui permet de determiner le type de la variable.
ci-dessous les types de valeurs:

  • String
  • Number
  • Boolean
  • Null
  • Object
  • undefined
NB: On peut utiliser la fonction prédéfinie typeof pour déterminer le type de valeur ou de variable.
Voir javascript.html

3.5 Les constantes

Une constante est un conteneur tout comme la variable mais sa valeur ne change jamais.
Exemple: PI, le nom d'un pays, la date de naissance d'une personne ...
Voir javascript.html

9. Les boucles

9.1 la boucle for

L'instruction for permet de créer une boucle avec 3 éléments séparés par des points virgules et entourés par des parentheses suivi d'une pair d'accolades contenant l'ensemble des instructions à exécuter.
Le premier élément est la condition de départ, le second la condition d'arret et le dernier le pas.

Les instructions break et continue sont très importantes pour manipuler les boucles.
le break permet d'arreter la boucle à l'itération ciblée.

9.2 la boucle for in

l'instruction for in permet d'iterer les indices associés aux valeurs d'un tableau.

9.3 la boucle for of

9.4 la boucle for each

l'instruction forEach permet d'iterer en meme temps les indices et les valeurs d'un tableau.

9.5 la boucle while

L'instruction while permet de créer une boucle, elle teste d'abord une condition et execute les instructions tant que la condition est vérifiée.
Exemple: Voir javascript.html

9.6 la boucle do while

L'instruction do while permet de créer une boucle qui s'exécute tantque la condition est vérifiée. elle exécute d'abord les instructions ensuite teste la condition ce qui veut dire qu'avec le do while les instructions s'exécutent au moins une fois.
Exemple: Voir javascript.html

11. Les fonctions

11.1. Definition:

Une fonction est un groupe de ligne de code de programmation, écrites par un concepteur et destinées à exécuter une tâche bien spécifique.
On appelle fonction un sous-programme qui permet d’effectuer un ensemble d’instructions par simple appel de la fonction dans le corps du programme principal.
Une fonction est un groupe de ligne(s) de code de programmation destiné à exécuter une tâche bien spécifique et que l’on pourra, si besoin utiliser à plusieurs reprises.
En JavaScript, il existe deux types de fonctions : Les fonctions prédéfinies et les fonctions déclarées.

11.2. Les fonctions prédéfinies:

En Javascript il existe un certains nombres de fonctions prédifinies, elles sont souvent associées à un objet c'est pour ce là qu'on les appelle méthodes.
Les fonctions alert(), prompt(), confirm ... sont associées à l'objet window, donc elles sont des méthodes de l'objet window.
Les fonctions pop(), push(), shift()... sont quant à elles des métodes de l'objet Array.

11.3. Les fonctions déclarées:

Elles sont les fonctions mises en place par le programmeur selon les besoins de l'application pour effectuer une tache bien précise.

Syntaxe:

function nom_de_la_fonction(les arguments){ --- les instructions --- }

Fonctions fléchées syntaxe:

(les arguments) => { --- les instructions --- }

Appel d'une fonction:

Pour appeler la fonction, il faut bien sûr qu’elle ait été définie auparavant. Par conséquent, il est conseillé de commencer la définition de la fonction et les appels dans le corps de page. L’appel d’une fonction se fait par son identifiant suivi impérativement de parenthèses, lesquelles peuvent contenir les arguments à faire passer.
Peut avoir lieu à n’importe quel endroit de la page :
- dans d’autres fonctions,
- dans le corps de la page.

12. Les objets

JavaScript est conçu autour d'un paradigme simple, basé sur les objets. Un objet est un ensemble de propriétés et de méthodes. Une propriété est une association entre un nom (aussi appelé clé) et une valeur. La valeur d'une propriété peut être une fonction, auquel cas la propriété peut être appelée « méthode ». En plus des objets natifs fournis par l'environnement, il est possible de construire ses propres objets.
Javascript n'utilisait pas d'instruction de class comme d'autres langages de programmation orientée Objet comme java, c++, ... Il utilisait les fonctions comme instruction de class.
Mais depuis ES6 (ECMAScript) le standard qui définit les normes de javascript publié en 2015, l'instruction class est présente et permet de construire des objets.
- A l'intérieur de la fonction ou de la class, on utilise: this.propriété.
- A l'extérieur de la fonction ou de la class, on utilise: instance.propriété.
Exemple: voir javascript.html

15. Les API

API signifie Application Programming Interface et est un concept qui n'est ni limité ni spécifique à JavaScript , mais qui est utilisé dans presque tous les langages d'application Web. En tant que développeur Web, vous devez connaître l'API.
En tant que concept, l'API existe depuis plus de 50 ans, mais ces dernières années, elle est devenue très populaire et a décrit une méthode différente pour créer nos applications.
Le but de l'API est de permettre la communication entre les applications en tant qu'intermédiaire , les deux applications pouvant avoir été créées avec des outils et des technologies différents. Cela peut être réalisé en utilisant des normes API telles que REST, SOAP, qui décrivent des protocoles ou des ensembles de règles que le client (qui doit appeler) et le serveur (qui répond aux demandes du client) doivent suivre. Ces normes sont essentielles à la communication. Ces normes décrivent à quoi devrait ressembler une requête et le format dans lequel le client doit attendre une réponse du serveur.
REST (Representational State Transfer) est un ensemble de principes architecturaux. SOAP (Simple Object Access Protocol) est un protocole officiel géré par le W3C (World Wide Web Consortium). La principale différence entre les deux est que SOAP est un protocole, REST non. En général, les API suivent l'approche REST ou SOAP en fonction de leur utilisation et des préférences du développeur.

API REST
Une API web est dite "REST" ou “Restful” si elle a les caractéristiques suivantes:
- Les requêtes sont envoyées sous forme de requêtes HTTP:
- Méthodes HTTP: GET, POST, PUT, PATCH, DELETE...
- Un Endpoint ou une route: http://example.com/api/ressources
- Les requêtes doivent être envoyées avec un MIME*/Content-type spécifique tel: JSON, XML, HTML, etc.

Contraintes d’une API REST
Une API Restful doit respecter certaines contraintes d’architecture:
1. Archi client-serveur: chacun évolue indépendamment.
2. Serveur sans états (stateless): pas de sessions.
3. Utilisation du cache: le client sait combien de temps il peut garder les données qu’il reçoit avant expiration.
4. Une interface uniforme: un seul id, contient l’URL des ressources suivantes...
5. Archi en couches: Réduire la complexité de l’architecture globale de l’API.
6. Code à la demande: étendre les fonctionnalités du client.


Ressources
En REST tout est resources, on interagit avec les ressources à travers différents routes (URLs):
- http://example.com/products
- http://example.com/products/1
- http://example.com/products/1/description
Pour interagir avec une ressource, il suffit d’utiliser la méthode HTTP e'"adéquate :
- GET http://example.com/products/1 (Récupérer)
- POST http://example.com/products (Créer)
- PUT ou PATCH http://example.com/products/1 (Mettre à jour)
- DELETE http://example.com/products/1 (Effacer)

Fetch
L’API Fetch comporte une seule fonction, concise et facile à utiliser fetch('url', 'options');
● fetch prend en paramètre l’URL de la ressource que vous voulez consulter et des options si besoin.
● Elle retourne une Promesse.

Les Promesses:
Fetch retourne une promesse qui peut être dans un des états suivants:
● pending: état initial, promesse non-exécutée
● fulfilled: la promesse a été exécutée correctement
● rejected: la promesse a rencontrée une erreur
On attache un gestionnaire à une promesse via .then()
fetch('images.txt').then(response => console.log(response.status));

API SOAP : Simple Object Access Protocol
SOAP est un protocole standard initialement conçu pour que des applications développées avec différents langages sur différentes plateformes puissent communiquer. Comme il s'agit d'un protocole, il impose des règles intégrées qui augmentent la complexité et les coûts, ce qui peut ralentir le chargement des pages. Cependant, ces standards assurent la conformité et sont ainsi privilégiés pour certains scénarios d'entreprise. Les standards de conformité intégrés incluent la sécurité, l'atomicité, la cohérence, l'isolement et la durabilité (ACID), un ensemble de propriétés qui permet d'assurer des transactions de base de données fiables.
Voici les principales spécifications de services web :
WS-Security (Web Services Security) : spécification qui standardise la manière dont les messages sont sécurisés et transférés via des identifiants uniques appelés jetons.
WS-ReliableMessaging : spécification qui standardise la gestion des erreurs entre les messages transférés par le biais d'une infrastructure informatique non fiable.
WS-Adressing (Web Services Addressing) : spécification qui ajoute les informations de routage des paquets en tant que métadonnées dans des en-têtes SOAP, au lieu de les conserver plus en profondeur dans le réseau.
WSDL (Web Services Description Language) : décrit la fonction d'un service web ainsi que ses limites.

Lorsqu'une requête de données est envoyée à une API SOAP, elle peut être gérée par n'importe quel protocole de couches de l'application : HTTP (pour les navigateurs web), SMTP (pour les e-mails), TCP et autres. En revanche, suite à la réception de la requête, les messages SOAP doivent être renvoyés sous la forme d'un document XML, un langage balisé lisible aussi bien par les humains que par les machines. Une fois finalisée, une requête destinée à une API SOAP ne peut pas être mise en cache par un navigateur. Il n'est donc pas possible d'y accéder plus tard sans la renvoyer vers l'API.

A savoir:

  • JSON: Javascript Object Notation est un format d'échange de données.
  • CORS: Cross-Origin Resource Sharing est un mécanisme qui permet à des ressources restreintes d'une page web d'etre récupérées par un autre domaine exterieur au domaine à partir duquel la première ressource a servie. Le CORS permet de prendre en charge des requetes multi-origines sécurisées et des transfert de données.
  • MIME: Multiporpose Internet Mail Extensions est un standard permettant d'indiquer la nature et le format d'un document.
